Framework-agnostic domain

Domain entities have no @Entity and don't depend on Spring. They are copied literally to the new microservice when the time comes.

Repository interfaces

Use cases only know the UserRepository interface. Today JPA implements it; tomorrow an HTTP client will. The use case never changes.

Custom policy interceptor

No Spring Security. An MVC Interceptor reads policies from Redis: method + path + effect (ALLOW/DENY). Configured in DB and applied without redeploy.

DDD + AI = 3-5 days per module

DDD gives AI the structure it needs to generate correct code. Without DDD, AI generates fast but coupled. With DDD, it generates fast and right.

multi-tenant by design

Database-level isolation.
Every tenant. Always.

Five isolated PostgreSQL schemas with Row-Level Security enforced at the database — not in application code. One tenant can never access another's data, by design.

⚠ Strict rule — no cross-schema JOINs · UUIDs as references only · no cross-schema FOREIGN KEYs → each module can have its own DB from day one
